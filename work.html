Pokedex

The Pokedex App is my first JavaScript project, designed to help me learn the fundamentals of JavaScript.
It showcases a list of 150 Pokemon, sourced from the external API (https://pokeapi.co/). When you click on a pokemon a modal window with the details such as: name, height, type and a photo of the Pokemon appears. In order to close the modal, you can click on the X on the top right corner, close on the bottom right corner, click outside the window or click the escape button.

Technologies Used:

  • HTML
  • CSS
  • JavaScript
  • jQuery
  • Bootstrap

This app demonstrates my proficiency in using these technologies to create an interactive and informative user experience.


Movie api

In this project I have built the server-side component of a “movies” web application, which provides users with access to information about different movies, directors and genres. Users are able to sign up, update their personal information and create a list of their favourite movies or remove a movie from their list of favourites.

Technologies Used:

  • Server-Side: Node.js, Express
  • Database: MongoDB
  • Middleware: body-parser (for reading data from requests), morgan (for logging)
  • Object Data Modeling: Mongoose
  • Authentication and Authorization: Included in the API
  • Data Validation: Implemented in the API logic
  • Testing: Postman
  • Deployment: GitHub, Heroku

Meet App

Meet App is a serverless, progressive web application (PWA) developed with React. It enables users to search for events in specific cities using the Google Calendar API.
The app offers various features like offline support, push notifications, and a responsive design. With a focus on high-quality code and adequate test coverage through a TDD approach, Meet App ensures scalability, availability, and a great user experience. Data visualization plays a significant role in the app, showcasing upcoming events by city and event genre popularity through charts. The integration of serverless functions allows for efficient backend management, scaling, and cost optimization.

Technologies Used:

  • Front-End: React
  • Backend: Serverless (AWS Lambda)
  • API Integration: Google Calendar API
  • Testing: Test-Driven Development (TDD) approach
  • Data Visualization: Charts (Number of upcoming events by city, popularity of event genres)
  • Features: Offline support, push notifications, add-to-home-screen prompt, responsive design

Chat App

The objective of this project was to develop a chat app for mobile devices using React Native. The app enables users to engage in chat conversations, share images, and exchange location information.

Technologies Used

  • Framework: React Native with Expo
  • Database: Google Firestore Database for storing chat conversations
  • Authentication: Google Firebase authentication for anonymous user authentication
  • Storage: Firebase Cloud Storage for image storage
  • Map Integration: Sending and displaying location data using a map view
  • Library: Utilized the Gifted Chat library for chat interface and functionality
